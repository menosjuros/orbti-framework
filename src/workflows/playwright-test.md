<purpose>
Execute automated acceptance tests using Playwright CLI based on the acceptance criteria from the current plan's SUMMARY.md. Generates a .spec.ts test file, runs it, and collects screenshots/traces as evidence.
</purpose>

<when_to_use>
- User runs /orbit:test and Playwright is detected in the project
- User explicitly requests automated testing via Playwright
- After BUILD phase completes on a web/UI feature
</when_to_use>

<prerequisites>
Playwright must be available in the project:
```bash
npx playwright --version 2>/dev/null
```
If not found, suggest: `npm init playwright@latest` and exit this workflow.
</prerequisites>

<process>

<step name="read_config">
Read `.orbit/config.md` testing block:

```yaml
testing:
  automated: true
  type: ui          # or both
  base_url: http://localhost:3000
  evidence:
    video: true       # record video?
    screenshot: true  # capture screenshots?
    logs: true
```

Store `evidence_video`, `evidence_screenshot`, `base_url` from config.

If config missing or `automated: false` → route to manual verify-work.md.
</step>

<step name="detect_playwright">
Check if Playwright is available:
```bash
npx playwright --version 2>/dev/null && echo "FOUND" || echo "NOT_FOUND"
```

If NOT_FOUND:
```
Playwright not detected in this project.

To enable automated testing:
  npm init playwright@latest

Falling back to manual testing.
```
Then route to manual verify-work.md workflow.

If FOUND: proceed.
</step>

<step name="read_acceptance_criteria">
Read the SUMMARY.md identified in the test scope.

Extract:
1. **Acceptance Criteria** — each AC with Given/When/Then
2. **Base URL** — from PROJECT.md or ask user
3. **User-facing routes** — pages/flows affected

If base URL not found in PROJECT.md, ask:
```
What is the base URL for testing?
(e.g., http://localhost:3000)
```
Store as `base_url`.
</step>

<step name="generate_spec">
Generate a Playwright spec file at:
`.orbit/phases/{phase-dir}/{plan}-e2e.spec.ts`

Template for each AC (Given/When/Then → test block):

```typescript
import { test, expect } from '@playwright/test';

// Generated by ORBIT /orbit:test
// Source: {path-to-SUMMARY.md}
// Date: {timestamp}

// Video is recorded automatically via playwright.config.ts (video: 'on')

test.describe('{Plan Name} — Acceptance Tests', () => {

  // AC-1: {criteria title}
  // Given {precondition}
  // When {action}
  // Then {outcome}
  test('AC-1: {criteria title}', async ({ page }) => {
    // Given
    await page.goto('{base_url}{relevant-path}');

    // When
    // {translated action steps}

    // Then
    // {assertions based on outcome}
    await expect(page).toHaveTitle(/{expected}/);

    // Evidence screenshot
    await page.screenshot({
      path: '.orbit/phases/{phase-dir}/evidence/{plan}-AC-1-pass.png',
      fullPage: true
    });
  });

});
```

Also generate/update `playwright.config.ts` based on config evidence settings:

```typescript
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    video: '{evidence_video ? "on" : "off"}',           // from config: evidence.video
    screenshot: '{evidence_screenshot ? "on" : "off"}', // from config: evidence.screenshot
    trace: 'on',                                        // always on for debugging
  },
  outputDir: '.orbit/phases/{phase-dir}/evidence',
});
```

If `playwright.config.ts` already exists, update only `video`, `screenshot`, and `trace` under `use:` — do NOT overwrite other settings.

**Mapping rules (Given/When/Then → Playwright):**

| Given/When/Then | Playwright equivalent |
|----------------|----------------------|
| user navigates to X | `page.goto(url)` |
| user clicks X | `page.click(selector)` |
| user fills X with Y | `page.fill(selector, value)` |
| user sees X | `expect(page.locator(selector)).toBeVisible()` |
| X contains text Y | `expect(page.locator(selector)).toContainText(Y)` |
| page title is X | `expect(page).toHaveTitle(X)` |
| URL is X | `expect(page).toHaveURL(X)` |
| element X exists | `expect(page.locator(selector)).toBeVisible()` |

**On ambiguous criteria:** generate the test scaffold with a `// TODO: adjust selector` comment — do NOT skip the test.

Create evidence directory:
```bash
mkdir -p .orbit/phases/{phase-dir}/evidence
```
</step>

<step name="run_tests">
Run the generated spec:
```bash
npx playwright test .orbit/phases/{phase-dir}/{plan}-e2e.spec.ts \
  --reporter=list,html \
  --output=.orbit/phases/{phase-dir}/evidence \
  2>&1
```

Capture full stdout/stderr output.

Also run with traces on failure:
```bash
npx playwright test .orbit/phases/{phase-dir}/{plan}-e2e.spec.ts \
  --trace=on-first-retry \
  --output=.orbit/phases/{phase-dir}/evidence \
  2>&1
```
</step>

<step name="collect_evidence">
After run completes, collect:

1. **Exit code** — 0 = all pass, non-zero = failures
2. **Videos** — `.webm` files per test in evidence dir (full recording of each test)
3. **Screenshots** — `.png` per test
4. **HTML report** — `playwright-report/index.html`
5. **Traces** — `.zip` files in evidence dir (for debugging failures)

Parse test output for:
- Total tests run
- Passed count
- Failed count
- Each failed test name + error message

Videos are saved as:
`evidence/{test-name}/video.webm`

To play a video: open the `.webm` file in any browser or video player.
To view a trace: `npx playwright show-trace evidence/{test-name}/trace.zip`
</step>

<step name="write_evidence_report">
Create `.orbit/phases/{phase-dir}/{plan}-EVIDENCE.md`:

```markdown
# Test Evidence: {Plan Name}

**Date:** {timestamp}
**Runner:** Playwright {version}
**Spec:** `.orbit/phases/{phase-dir}/{plan}-e2e.spec.ts`
**Base URL:** {base_url}

## Results

| Test | AC | Status | Video | Screenshot |
|------|----|--------|-------|------------|
| AC-1: {title} | AC-1 | ✅ PASS | [video](evidence/AC-1/video.webm) | [screenshot](evidence/{plan}-AC-1-pass.png) |
| AC-2: {title} | AC-2 | ❌ FAIL | [video](evidence/AC-2/video.webm) | [screenshot](evidence/{plan}-AC-2-fail.png) |

**Passed:** N  **Failed:** N  **Total:** N

## Failures

### {test name}
**Error:**
\`\`\`
{error message from Playwright output}
\`\`\`
**Video:** `evidence/{test-name}/video.webm`
**Trace:** `evidence/{trace-file}.zip` — `npx playwright show-trace evidence/{trace-file}.zip`

## Verdict
{PASS / FAIL based on results}
```
</step>

<step name="report_to_user">
Display summary:

```
════════════════════════════════════════
PLAYWRIGHT TEST RESULTS
════════════════════════════════════════

Spec: .orbit/phases/{phase-dir}/{plan}-e2e.spec.ts
Run:  {timestamp}

  ✅  AC-1: {title}
  ✅  AC-2: {title}
  ❌  AC-3: {title}
      └─ {error snippet}

────────────────────────────────────────
Passed: N | Failed: N | Total: N

Evidence: .orbit/phases/{phase-dir}/evidence/
Report:   playwright-report/index.html
────────────────────────────────────────

Verdict: {ALL PASS ✅ | FAILURES FOUND ❌}
```

**If all pass:**
→ Route to /orbit:integrate

**If failures:**
→ Offer: "Run /orbit:plan-fix to create a fix plan for the failures"
→ UAT issues already logged to `.orbit/phases/{phase-dir}/{plan}-UAT.md`
</step>

<step name="log_failures_to_uat">
For each failed test, auto-log to UAT file using the same format as verify-work.md:

```markdown
### UAT-{NNN}: {test title}

**Discovered:** {date} via Playwright automated test
**Phase/Plan:** {phase}-{plan}
**Severity:** Blocker
**AC:** {AC reference}
**Description:** Automated test failed
**Error:** {playwright error message}
**Evidence:** `.orbit/phases/{phase-dir}/evidence/{screenshot}`
**Trace:** `.orbit/phases/{phase-dir}/evidence/{trace}.zip`
```
</step>

</process>

<output>
- `.orbit/phases/{phase-dir}/{plan}-e2e.spec.ts` — generated Playwright spec
- `.orbit/phases/{phase-dir}/evidence/` — screenshots and traces
- `.orbit/phases/{phase-dir}/{plan}-EVIDENCE.md` — evidence report
- `.orbit/phases/{phase-dir}/{plan}-UAT.md` — failures logged (if any)
- `playwright-report/index.html` — Playwright HTML report
</output>

<error_handling>
**Playwright not installed:**
- Suggest: `npm init playwright@latest`
- Fall back to manual testing

**Spec generation fails (ambiguous criteria):**
- Generate scaffold with TODO comments
- Run anyway — failing tests are still evidence

**Base URL unreachable:**
- Report: "Could not connect to {base_url}"
- Ask user to start the dev server first

**All tests fail with import errors:**
- Check if `@playwright/test` is installed
- Suggest: `npm install -D @playwright/test`
</error_handling>
