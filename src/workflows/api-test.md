<purpose>
Execute automated API acceptance tests using Playwright's request fixture. No browser needed. Tests HTTP endpoints against acceptance criteria, logs request/response pairs as evidence.
</purpose>

<when_to_use>
- Config has testing.type: api or both
- Project is a REST API, backend service, or microservice
- /orbit:test is called and automated: true
</when_to_use>

<process>

<step name="read_config">
Read `.orbit/config.md` testing block:

```yaml
testing:
  automated: true
  type: api          # or both
  api_base_url: http://localhost:3000/api
  evidence:
    logs: true
```

If `api_base_url` not set, ask:
```
What is the API base URL?
(e.g., http://localhost:3000/api)
```
</step>

<step name="read_acceptance_criteria">
Read the SUMMARY.md for the test scope.

Extract API-specific criteria:
- Endpoints mentioned (GET /users, POST /auth, etc.)
- Expected status codes
- Expected response shapes
- Authentication requirements
- Edge cases (404, 401, 422, etc.)

Map each AC to one or more API test cases.
</step>

<step name="generate_api_spec">
Generate spec at `.orbit/phases/{phase-dir}/{plan}-api.spec.ts`:

```typescript
import { test, expect } from '@playwright/test';

// Generated by ORBIT /orbit:test (API mode)
// Source: {path-to-SUMMARY.md}
// Date: {timestamp}
// Base URL: {api_base_url}

test.describe('{Plan Name} — API Acceptance Tests', () => {

  // AC-1: {criteria title}
  // Given {precondition}
  // When {HTTP call}
  // Then {expected response}
  test('AC-1: {criteria title}', async ({ request }) => {
    // Given — set up any required state
    // (auth token, seed data, etc.)

    // When
    const response = await request.get('{api_base_url}/endpoint');

    // Then
    expect(response.status()).toBe(200);
    const body = await response.json();
    expect(body).toMatchObject({
      // expected shape
    });

    // Log evidence
    console.log('AC-1 response:', JSON.stringify(body, null, 2));
  });

  test('AC-1 (error case): returns 404 when not found', async ({ request }) => {
    const response = await request.get('{api_base_url}/endpoint/nonexistent');
    expect(response.status()).toBe(404);
  });

});
```

**Mapping rules (Given/When/Then → Playwright request):**

| Given/When/Then | Playwright equivalent |
|----------------|----------------------|
| GET endpoint X | `request.get(url)` |
| POST to X with body Y | `request.post(url, { data: Y })` |
| PUT to X | `request.put(url, { data: body })` |
| DELETE X | `request.delete(url)` |
| with auth header | `request.get(url, { headers: { Authorization: 'Bearer ...' } })` |
| returns status N | `expect(response.status()).toBe(N)` |
| response contains field X | `expect(body).toHaveProperty('X')` |
| response matches shape | `expect(body).toMatchObject({...})` |
| response is array | `expect(Array.isArray(body)).toBe(true)` |

**Always generate error case tests:**
- 404 for non-existent resources
- 401/403 for missing/invalid auth (if auth is in scope)
- 422 for invalid payloads (if validation is in scope)

Create evidence directory:
```bash
mkdir -p .orbit/phases/{phase-dir}/evidence
```
</step>

<step name="run_api_tests">
Run the spec (no browser flags needed):
```bash
npx playwright test .orbit/phases/{phase-dir}/{plan}-api.spec.ts \
  --reporter=list,html \
  2>&1
```

Capture full stdout. API tests produce no video/screenshots — evidence is in logs.
</step>

<step name="collect_api_evidence">
For each test, capture:
1. **Status** — pass / fail
2. **HTTP status code** — from response
3. **Response body** — logged via `console.log` in spec
4. **Error message** — if test failed (assertion mismatch)
5. **Duration** — response time

Write log file `.orbit/phases/{phase-dir}/evidence/{plan}-api-log.json`:

```json
{
  "run": "{timestamp}",
  "base_url": "{api_base_url}",
  "tests": [
    {
      "name": "AC-1: {title}",
      "status": "pass",
      "duration_ms": 142,
      "http_status": 200,
      "response_preview": { "id": 1, "name": "..." }
    },
    {
      "name": "AC-2: {title}",
      "status": "fail",
      "duration_ms": 89,
      "http_status": 404,
      "error": "expect(received).toBe(expected) — Expected: 200, Received: 404"
    }
  ]
}
```
</step>

<step name="write_api_evidence_report">
Create `.orbit/phases/{phase-dir}/{plan}-EVIDENCE.md`:

```markdown
# API Test Evidence: {Plan Name}

**Date:** {timestamp}
**Runner:** Playwright {version} (API mode, no browser)
**Spec:** `.orbit/phases/{phase-dir}/{plan}-api.spec.ts`
**Base URL:** {api_base_url}

## Results

| Test | AC | Status | HTTP | Duration |
|------|----|--------|------|----------|
| AC-1: {title} | AC-1 | ✅ PASS | 200 | 142ms |
| AC-2: {title} | AC-2 | ❌ FAIL | 404 | 89ms |

**Passed:** N  |  **Failed:** N  |  **Total:** N

## Failures

### {test name}
**Expected HTTP:** 200  **Got:** 404
**Error:** `{assertion message}`

## Log
Full request/response log: `evidence/{plan}-api-log.json`

## Verdict
{ALL PASS ✅ | FAILURES FOUND ❌}
```
</step>

<step name="report_to_user">
Display:

```
════════════════════════════════════════
PLAYWRIGHT API TEST RESULTS
════════════════════════════════════════

Spec:     .orbit/phases/{phase-dir}/{plan}-api.spec.ts
Base URL: {api_base_url}

  ✅  AC-1: GET /users returns list          200  142ms
  ✅  AC-2: POST /users creates user         201   89ms
  ❌  AC-3: GET /users/:id returns user
      └─ Expected 200, got 404

────────────────────────────────────────
Passed: N | Failed: N | Total: N

Log:    .orbit/phases/{phase-dir}/evidence/{plan}-api-log.json
Report: playwright-report/index.html
────────────────────────────────────────

Verdict: {ALL PASS ✅ | FAILURES FOUND ❌}
```

If failures → offer `/orbit:plan-fix`
</step>

</process>

<output>
- `.orbit/phases/{phase-dir}/{plan}-api.spec.ts` — generated API spec
- `.orbit/phases/{phase-dir}/evidence/{plan}-api-log.json` — request/response log
- `.orbit/phases/{phase-dir}/{plan}-EVIDENCE.md` — evidence report
- `.orbit/phases/{phase-dir}/{plan}-UAT.md` — failures logged (if any)
- `playwright-report/index.html` — HTML report
</output>

<error_handling>
**API server not running:**
- Tests will fail with ECONNREFUSED
- Report: "Could not connect to {api_base_url} — start the server first"

**Auth required but not configured:**
- Generate tests with `// TODO: add auth token` comment
- Tests will return 401 — that's valid evidence too

**No endpoints in acceptance criteria:**
- Ask user: "Which endpoints should be tested?"
- Generate scaffold from their answer
</error_handling>
